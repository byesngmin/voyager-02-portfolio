제시해주신 프로젝트 맥락(SF/우주 테마의 포트폴리오)과 게임 기획자라는 직군 특성을 반영하여, 서사 구조를 매력적으로 전달하고 채용 담당자의 시선을 끝까지 유지할 수 있는 UX 라이팅 및 레이아웃 리서치 결과를 정리해 드립니다.

## 채용 담당자 읽기 패턴 핵심 인사이트 (3가지)

1. **F-패턴 스캐닝과 키워드 전진 배치 (Front-loading)**
   *   **인사이트:** 채용 담당자는 5~10분의 짧은 시간 동안 긴 글을 정독하지 않고, 좌측 상단에서 우측으로 훑는 'F-패턴'으로 스캐닝합니다.
   *   **대응:** 서사의 핵심 키워드(문제, 해결책, 성과)를 문단 첫 줄과 좌측에 배치하고, 볼드체나 하이라이트 컬러로 처리하여 훑어보기만 해도 전체 맥락이 파악되도록 설계해야 합니다.
2. **인지적 마찰 방지를 위한 정보의 점진적 공개 (Progressive Disclosure)**
   *   **인사이트:** 텍스트 벽(Wall of text)은 즉각적인 이탈(Bounce)을 유발합니다. 독자는 압도적인 텍스트 분량을 보는 순간 읽기를 포기합니다.
   *   **대응:** 원체험이나 철학은 2~3문단으로 짧게 청크(Chunk)를 나누고, 상세한 기획 의도나 부가 설명은 아코디언 UI, 모달창, 마우스 오버 툴팁 등을 통해 '원하는 사람만' 깊게 파고들 수 있도록 숨겨두어야 합니다.
3. **시각적 휴식과 앵커링 효과 (Visual Anchoring & White Space)**
   *   **인사이트:** 스크롤 피로도를 낮추고 이탈을 막으려면 뇌가 쉴 수 있는 공간과 시선을 다시 붙잡는 장치가 필요합니다.
   *   **대응:** 챕터가 전환될 때 의도적으로 여백(White space)을 넓게 확보하고, 강력한 인용구(Quote)나 시각화된 성과 수치를 앵커로 사용하여 스크롤을 멈추게(Stop-scroll) 만들어야 합니다.

## 우수 자기소개 페이지 레퍼런스 (URL + 기법 분석, 최소 5개)

1. **Bruno Simon** (`bruno-simon.com`)
   *   **기법 분석:** 3D WebGL을 활용해 사용자가 미니 자동차를 운전하며 포트폴리오를 탐색하게 하는 극단적 게이미피케이션. '텍스트'가 아닌 '경험'으로 자신의 정체성과 기술력을 완벽히 증명한 사례입니다.
2. **Jason Yuan** (`jasonyuan.design`)
   *   **기법 분석:** 에세이 형태의 서사적 레이아웃. 거대한 타이포그래피로 오프닝 은유를 제시하고, 세로 스크롤과 가로 스크롤을 혼합하여 글을 읽는 호흡과 템포를 UI 레벨에서 능동적으로 통제합니다.
3. **Brittany Chiang** (`brittanychiang.com`)
   *   **기법 분석:** 고정형 사이드바(Sticky Navigation)를 적극 활용. 화면 우측에서 자기소개가 진행되는 동안, 좌측 인디케이터가 스크롤 위치에 맞춰 활성화(Scroll Spy)되어 채용 담당자가 현재 어느 섹션을 읽고 있는지 명확한 길잡이를 제공합니다.
4. **Damien Bel** (`damienbelgames.com`)
   *   **기법 분석:** 게임 기획자로서 자신의 철학("시스템과 내러티브의 결합")을 대비되는 색상과 레이아웃으로 명확히 구분. 복잡한 텍스트 대신 기획 산출물(플로우차트, 엑셀 밸런스 시트)을 이미지 갤러리로 삽입하여 실무 역량을 시각적으로 어필합니다.
5. **Noemi Seoane** (`noemiseoane.com`)
   *   **기법 분석:** 레벨 디자이너의 특성을 살려, 스크롤을 내릴 때마다 배경 이미지가 전환되며 새로운 세계(Scene)가 열리는 듯한 패럴랙스 효과를 줍니다. 텍스트 스토리에 맞춰 배경이 시각적 메타포를 제공하여 몰입감을 극대화합니다.

## 타이포그래피 계층 구조 제안 (챕터별)

프로젝트에 내재된 우주/신호(Voyager, Signal Archive) 메타포를 극대화하기 위해 **SF 디스플레이 폰트와 깔끔한 한글 고딕의 조합**을 제안합니다.

*   **H1 (오프닝 은유):** `Space Grotesk` (영문) + `Pretendard Black` (한글)
    *   **스타일:** 4rem 이상의 거대한 크기, 자간(Tracking)을 좁게 하여 영화 포스터 타이틀처럼 압도적인 임팩트.
*   **H2 (챕터 헤딩):** `Orbitron` 또는 `Rajdhani` (영문 번호/코드) + `Pretendard Bold` (한글)
    *   **스타일:** 2.5rem, 영문은 대문자로 처리. 제목 좌측에 네온 컬러의 버티컬 라인(|)을 두어 터미널 프롬프트 느낌 강조. (예: `CH.01 | 나의 원체험`)
*   **Quote (철학 진술/인용):** `JetBrains Mono` 또는 명조체(`KoPub 바탕`)
    *   **스타일:** 1.5rem, 이탤릭체 적용. 우주에서 수신된 '신호(Signal)'나 핵심 메시지의 느낌을 연출하기 위해 본문 폰트와 이질감을 줌.
*   **Body (본문 텍스트):** `Pretendard Regular`
    *   **스타일:** 1.125rem. 가독성을 위해 줄 간격(Line-height) 1.6~1.8 유지. 시선 피로를 줄이기 위해 텍스트 폭(Max-width)을 한 줄당 65~75자로 엄격히 제한.
*   **Highlight (강조 텍스트):**
    *   **스타일:** 텍스트 배경을 옅은 네온 블루/앰버 색상으로 칠하거나 밑줄을 점선(Dashed) 처리 후 폰트 웨이트를 올림(SemiBold).

## 챕터 흐름 유도 UI 패턴 목록 (5가지)

1. **스크롤 진행도 타임라인 (Scroll Progress Sidebar):** 화면 좌측 또는 우측에 희미한 세로선을 긋고, 각 챕터를 나타내는 점(Node)을 배치합니다. 스크롤할 때마다 현재 챕터에 불이 들어와 '얼마나 더 읽어야 하는지' 명확한 동기를 부여합니다.
2. **스티키 서브헤딩 (Sticky Subheadings):** 본문을 읽어 내려갈 때, 해당 챕터의 소제목이 화면 상단에 찰칵(Sticky) 붙어 고정됩니다. 긴 텍스트를 읽는 중에도 현재 문맥을 잊지 않게 해줍니다.
3. **터미널 스타일 콜아웃 박스 (Terminal-style Callouts):** 텍스트 벽 중간에 검은색 배경과 초록/파랑 모노스페이스 폰트를 사용한 '박스 UI'를 삽입합니다. 기획자의 '시스템 로그'나 '기획 노트'처럼 보이게 하여 시각적 환기를 유도합니다.
4. **스크롤 연동 페이드 인 (Scroll-Triggered Fade-in):** 한 페이지에 모든 텍스트를 띄우지 않습니다. 스크롤 뷰포트에 도달할 때마다 각 문단이나 오브젝트가 아래에서 위로 부드럽게 나타나게 하여 인지적 부담을 대폭 줄입니다.
5. **인터랙티브 각주 툴팁 (Interactive Footnote Tooltips):** 부가적인 설명(예: 특정 게임 시스템 용어, 과거 개발 비화)은 본문에 바로 쓰지 않고 점선 밑줄로 처리합니다. 마우스를 올리면 툴팁이나 말풍선으로 정보가 제공되어 본문의 흐름을 끊지 않습니다.

## 성과 수치 강조 패턴 (3가지)

1. **카운터 타이머 애니메이션 (Counter Animation):**
   *   숫자가 들어간 섹션이 화면에 노출되는 순간, `0`에서부터 실제 수치(예: `150%`)까지 빠르게 올라가는 애니메이션을 적용합니다. 동적인 움직임으로 채용 담당자의 시선을 즉각적으로 잡아끕니다.
2. **거대 스케일 대비 (Massive Number Scale):**
   *   수치를 설명 텍스트와 분리합니다. 숫자는 3rem~4rem 이상의 거대한 크기로, 하단의 라벨(예: "리텐션 증가율")은 1rem의 작은 크기로 배치하고 주변 여백을 넓게 비웁니다. 수치 자체가 하나의 그래픽 요소로 작동하게 합니다.
3. **네온 글로우 데이터 카드 (Neon Glow Stat Card):**
   *   단순 텍스트가 아닌 어두운 배경의 독립된 카드(Card) 형태로 성과를 배치합니다. 카드의 테두리나 숫자 자체에 은은한 네온 빛(Glow) 효과를 주어, 이 페이지에서 가장 핵심적인 '데이터'임을 직관적으로 인지시킵니다.
